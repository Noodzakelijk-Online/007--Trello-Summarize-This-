# No Vanity Work Rule

Date: 2026-07-23 (Phase 090)

## Policy Statement

1. **No Superfluous Refactoring:** Code modifications must solve documented bugs, harden security, or fulfill phase contract requirements.
2. **No Fake Completeness:** Unimplemented features (e.g. background workers, SaaS billing) are documented as Missing or Blocked, never claimed as finished without code evidence.
3. **Verified Deliverables:** Every phase marked `Implemented` must be backed by reproducible test output or verified documentation.
