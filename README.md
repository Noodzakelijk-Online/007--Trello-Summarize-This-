# Summarize This

Summarize This is a static Trello Power-Up for reviewing a card's context and producing an operational summary. It works locally by default and can use OpenAI, Google AI, or Anthropic when a member explicitly configures an AI provider or a protected backend proxy.

## What It Does

- Adds a `Summarize This` card button and a compact card-detail status badge.
- Reads the available card title, description, list, members, labels, checklist, comments, activity, and attachment metadata.
- Produces a local fallback summary when no AI provider is configured.
- Produces an evidence-aware AI summary with confidence and review signals when AI is enabled.
- Keeps analysis runs, feedback, review state, and cost records in member-private Power-Up storage.
- Provides copy formats and a structured JSON download for completed analysis runs.
- Supports optional, bounded text extraction from HTTPS text-like attachments. It is off by default and does not follow redirects.

The confidence value is a review signal, not a guarantee. Check summaries before making decisions or posting changes to Trello.

## Install The Power-Up

1. Publish this repository to a public HTTPS static host. GitHub Pages can serve the repository root directly.
2. Open `trello-setup.html` from that host. It generates the exact connector, policy, icon, and manifest URLs for Trello's Power-Up admin page.
3. In Trello Power-Up admin, save the generated iframe connector URL.
4. Add the Power-Up to a board and open a card.

For this repository's GitHub Pages deployment, the connector is:

`https://noodzakelijk-online.github.io/007--Trello-Summarize-This-/connector.html`

Use the versioned connector URL generated by `trello-setup.html` after a deployment. The current release uses `?v=20260719.1` to avoid Trello loading a cached connector page.

## AI And Privacy

AI is optional. In local-only mode, no card context is sent to an AI provider.

- Direct mode stores provider keys only in Trello member-private Power-Up storage for the current member.
- Proxy mode keeps provider keys on the server. The included Cloudflare Worker requires exact `ALLOWED_ORIGINS` values and uses the configured provider model unless an administrator explicitly allowlists other models.
- Sensitive-card approval can require user confirmation before card context leaves the browser.
- Proxy endpoints must be public HTTPS URLs. `http://localhost` and `http://127.0.0.1` are allowed only for local development. Private-network endpoints are rejected.

See [proxy/README.md](proxy/README.md) for the Worker deployment and security boundary.

## Windows Installer

Build the Windows 11 installer from PowerShell:

```powershell
npm.cmd run build:windows-installer
```

The result is `dist/windows-installer/SummarizeThisSetup.exe`. The installer verifies its embedded payload before extracting it, installs under the current user's local app data, and adds Start Menu shortcuts for the local preview and Trello configuration helper.

The local preview is not a Trello connector host. Trello requires a public HTTPS connector URL.

## Development

```powershell
npm.cmd test
npm.cmd run analyze:resources
```

`npm.cmd test` covers the summarizer, ledger retention, deployment helpers, proxy safeguards, attachment URL handling, and installer metadata. `npm.cmd run analyze:resources` reports the initial popup payload separately from the deferred attachment extractor.

The main runtime files are:

- `connector.html` and `connector.js`: Trello capability registration.
- `popup.html`: card analysis UI and Power-Up interactions.
- `settings-powerup.html`: member-private settings.
- `summarizer-core.js`: normalization, prompt construction, local analysis, and safety rules.
- `card-intelligence-ledger.js`: analysis ledger, evidence, review, and export data.
- `attachment-processor.js`: optional bounded text extraction.
- `trello-setup.html`: deployment and Power-Up admin helper.

## Release Checks

Before publishing a release:

```powershell
npm.cmd test
npm.cmd run analyze:resources
npm.cmd run build:windows-installer
```

After publishing to GitHub Pages, update Trello Power-Up admin with the current versioned connector URL and verify a card button, badge, settings popup, and local summary on a QA card.
