        Admin autofill helper

        Trello creates a new app in two stages. Drag the helper to your bookmarks bar, open the New App form, select the Workspace yourself, then click the helper to fill the visible text fields. After manually creating the app, open its edit page and run the helper again for matching metadata and capability checkboxes. It never saves or submits either page.

        Summarize This admin autofill
        • Review every populated Trello admin field before saving.
        • Select the owning Trello Workspace manually; the helper never changes Workspace ownership.
        • Enter a monitored Email and Support contact before creating the app.
        • If the helper reports missing fields, run it again from the app edit page or copy those values manually from the admin values section.
        • Use the generated HTTPS connector URL, not a local Windows address.
        • If Trello changes its admin page labels, use the copied admin values manually.
